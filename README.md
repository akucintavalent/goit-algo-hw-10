# goit-algo-hw-10

Домашня робота до тем 9 та 10: жадібні алгоритми, динамічне програмування та метод Монте-Карло.

## Завдання 1. Видача решти

У файлі `task1_coins.py` реалізовано дві функції для набору монет `[50, 25, 10, 5, 2, 1]`:

- `find_coins_greedy(amount)` - жадібний алгоритм, який спочатку бере найбільші доступні номінали.
- `find_min_coins(amount)` - алгоритм динамічного програмування, який знаходить мінімальну кількість монет для заданої суми.

Для суми `113` обидва алгоритми повертають:

```python
{50: 2, 10: 1, 2: 1, 1: 1}
```

### Порівняння ефективності

Жадібний алгоритм проходить лише по списку номіналів монет. Оскільки кількість номіналів фіксована, його складність становить `O(k)`, де `k` - кількість номіналів. Для цього набору монет результат жадібного алгоритму є оптимальним.

Алгоритм динамічного програмування перевіряє всі проміжні суми від `1` до `amount` і для кожної суми перебирає всі номінали. Його складність становить `O(amount * k)`, а також потрібна додаткова пам'ять `O(amount)`.

Приклад запуску для великої простої суми `99991`:

```text
Greedy time for 99991: 0.00000300 seconds
Dynamic programming time for 99991: 0.04852310 seconds
```

Висновок: для великих сум жадібний алгоритм значно швидший і простіший. Динамічне програмування є універсальнішим, бо може знайти оптимальне рішення для довільних наборів монет, де жадібний підхід не завжди працює оптимально.

## Завдання 2. Інтеграл методом Монте-Карло

У файлі `task2_monte_carlo.py` обчислюється визначений інтеграл функції:

```python
f(x) = -(x ** 2) + 4 * x + 3
```

на проміжку `[0, 4.5]`.

На цьому проміжку функція невід'ємна, тому метод Монте-Карло реалізовано через випадкові точки у прямокутнику з висотою `max_y = 7`. Частка точок, які потрапили під графік, множиться на площу прямокутника:

```text
integral = hits / samples * (b - a) * max_y
```

Для перевірки результат порівнюється з аналітичним значенням та результатом `scipy.integrate.quad`.

Приклад запуску:

```text
Analytical result: 23.625
Scipy result: 23.625
Scipy absolute error estimate: 2.6229018956769323e-13
Monte Carlo result: 23.586570000000002
Difference: 0.03842999999999819
```

Аналітичний результат:

```text
integral from 0 to 4.5 of (-x^2 + 4x + 3) dx =
(-x^3 / 3 + 2x^2 + 3x) | 0..4.5 = 23.625
```

Висновок: метод Монте-Карло дає наближений результат, який близький до точного значення та результату `quad`. Невелика різниця пояснюється випадковістю вибірки. Зі збільшенням кількості випадкових точок точність зазвичай покращується, але час виконання також зростає.
